# System Architecture

## The "Factory Floor" Concept
This system is designed as an automated video processing pipeline ("Conveyor Belt") that operates asynchronously to process raw screencasts and voiceovers generated by producers.

## Folder Hierarchy (Google Shared Drives)
The crawler strictly enforces this 3-level folder structure. If the structure is violated, the crawler skips the folder.

```
📁 Uploads Shared Drive
 └── 📁 Producer (e.g. Eli, Ad, Paula)
      └── 📁 Software (e.g. Gemini, ChatGPT)
           └── 📁 Date (YYYY-MM-DD)
                ├── raw_video.mp4  (2x speed screencast)
                └── raw_audio.mp3  (1x speed voiceover)
```

## Night Shift Workflow
Triggered by: `POST /api/process-batch`

1. **Crawl Phase**: Recursively lists all `YYYY-MM-DD` folders in the Uploads drive.
2. **Matching**: Pairs `video.mp4` with `audio.mp3` based on exact matching base filenames.
3. **Smart Skip (Timestamp Check)**: 
   - Checks the destination YouTube Drive to see if a finished version exists.
   - If `Uploads_Video.createdTime <= YouTube_Finished_Video.createdTime`, it skips processing.
4. **Smart Reprocessing**:
   - If `Uploads_Video.createdTime > YouTube_Finished_Video.createdTime`, the crawler realizes the producer uploaded a newer edit.
   - It bypasses Smart Skip, processes the video, and appends `_v2` to the output.
5. **FFmpeg Engine (`libx264`)**:
   - `setpts=2.0*PTS`: Slows the 2x video down to 1x to match the audio.
   - `scale=1920:1080`: Normalizes resolution.
   - `yuv420p` & `faststart`: Prepares the video for web streaming.
6. **Deep Cleanup (The Purge)**:
   - When a video is "Smart Skipped" but has been sitting in the Uploads Drive for > 48 hours, it is moved to the Google Drive **Trash** to save space.

## Alphabetical Output (The Golden Rule)
To ensure everything processes alphabetically, the application relies on `/api/drive-files-metadata` to sort folders/files *before* submitting them to the job queue or appending them to Google Sheets.

## Filename Conventions
- Input: `[Topic].mp4`
- Output: `[Topic]_[Producer]_FINISHED.mp4`
- Reprocessed Output: `[Topic]_[Producer]_vX_FINISHED.mp4`
- Flagged Output: `REVIEW_[Topic]_[Producer]_FINISHED.mp4` (Flagged if video and audio durations diverge by > 2 seconds).
